# Семестровий проєкт: Статистичний аналіз потокових даних (Histogram)

## 1. Про що наш проєкт?
Ми розробили програму для обробки великих масивів даних за допомогою структури **Histogram**. 
Головна ідея в тому, щоб аналізувати інформацію «на льоту» (режим Streaming Processing). 
На відміну від звичайних методів, наша програма не зберігає всі мільйони чисел у пам'яті, 
а одразу розподіляє їх по інтервалах.

Це робить систему дуже швидкою: додавання нового числа завжди займає мінімум часу ($O(1)$), 
а обсяг використаної пам'яті залежить тільки від кількості стовпців гістограми ($O(M)$), 
а не від кількості вхідних даних.

## 2. Математична логіка та аналіз
В основі проєкту лежить математична статистика для згрупованих даних. 
Оскільки ми не зберігаємо самі числа, усі обчислення ми проводимо, 
спираючись на середини інтервалів та їхні частоти.

**Наші інструменти аналізу:**
**Базова статистика:** Програма рахує середнє значення, медіану та дисперсію, 
що дає повне уявлення про «центр» та розкид даних.
**Форма розподілу:** Ми додали розрахунок асиметрії та ексцесу. Це допомагає зрозуміти, 
наскільки розподіл даних «скошений» або наскільки гострим є його пік порівняно з ідеальним випадком.
**Перевірка гіпотез:** Вагома частина — **перевірка критерію Пірсона ($\chi^2$)**. 
Ми реалізували алгоритми, які перевіряють, чи схожі ваші дані на нормальний, експоненціальний, 
рівномірний розподіли або на розподіли Пуассона чи Бернуллі.

## 3. Як усе влаштовано (Технічний стек)
Ми вирішили поєднати два підходи, створивши гібридне ядро:
**Низький рівень (C99):** Вся базова логіка написана на чистому C для максимальної швидкості.
**Об'єктний рівень (C++11):** Зручний інтерфейс, класи та обробка помилок реалізовані на C++.
**Збірка та універсальність:** Завдяки **CMake** проєкт легко запускається і на Windows 
(через Visual Studio), і на Linux.

## 5. Запуск проєкту
Ми додали скрипти автоматизації, щоб проєкт можна було зібрати однією командою:
На Windows достатньо запустити `build.bat`.
На Linux/macOS — скрипт `build.sh`.

Це автоматично скомпілює код та запустить тести, які ми підготували для перевірки точності розрахунків.
